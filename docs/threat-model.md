# 威胁模型 v0.1

## 系统边界

第一版系统包含：

- 用户
- FastAPI 后端
- 内存数据库
- 文档资产
- Bearer Token 鉴权

后续版本会增加：

- RAG 检索
- 向量数据库
- LLM API
- Agent 工具调用
- 自动化红队测试

## 资产清单

| 资产 | 风险 |
|---|---|
| 用户账号 | 弱口令、撞库、越权 |
| Bearer Token | 泄露、伪造、重放 |
| 文档内容 | 跨用户泄露、跨租户泄露、敏感信息泄露 |
| 租户 ID | 租户隔离失效 |
| 管理员接口 | 权限过大、未授权访问 |

## 信任边界

| 边界 | 说明 |
|---|---|
| 浏览器到 API | 所有请求都不可信 |
| API 到文档存储 | 必须写入 owner_id 和 tenant_id |
| 普通用户到文档 | 只能访问自己的文档 |
| 管理员到文档 | 可访问所有文档，但必须鉴权 |

## 当前重点风险

### API 越权

普通用户通过修改 `document_id` 访问其他用户文档。

预期安全结果：

- 返回 `403 Forbidden`
- 不返回文档标题和内容

### Token 伪造

攻击者修改 Token 中的用户 ID 或角色。

预期安全结果：

- HMAC 签名校验失败
- 返回 `401 Unauthorized`

### 管理员接口未授权

普通用户访问 `/admin/documents`。

预期安全结果：

- 返回 `403 Forbidden`

## 后续 RAG 风险

第二版接入 RAG 后，需要重点验证：

- 检索是否按 `tenant_id` 过滤
- 检索是否按 `owner_id` 或授权关系过滤
- 恶意文档是否能控制模型输出
- 模型是否会泄露系统提示
- 模型是否会输出其他租户文档内容
